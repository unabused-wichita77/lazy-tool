## 🚀 What lazy-tool does

lazy-tool helps you find local MCP tools before an agent tries to use them. It keeps your prompt smaller, cuts down on tool noise, and sends requests to the right local tool.

Use it when you want to:

- search local tools first
- route tasks to the best MCP tool
- keep prompts short
- reduce repeated tool lists
- work with local-first agent setups
- use a simple interface for tool discovery

## 📂 How to use it

1. Start lazy-tool.
2. Let it scan or load your local MCP tools.
3. Search for the tool or task you want.
4. Review the best match.
5. Send the request to the local tool.
6. Use the result in your agent flow.

If you work with multiple tools, search by task name, tool name, or short keywords. That helps you find the right one fast.

## 🛠️ Troubleshooting

If the app does not start:

- check that you downloaded the Windows file
- make sure the file finished downloading
- extract ZIP files before opening the app
- try running the app as administrator
- update Windows and try again
- download the latest release if you used an older file

If Windows blocks the file:

- open the file’s properties
- look for an unblock option
- allow the file if you trust the release page source
- try again after extraction if the file was in a ZIP folder

If the app opens but tools do not appear:

- make sure your local MCP tools are installed
- refresh the tool list
- check the local tool path
- restart the app
- use the latest release from the download page
